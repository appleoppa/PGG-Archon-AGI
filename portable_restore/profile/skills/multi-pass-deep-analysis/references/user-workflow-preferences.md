# 用户工作流偏好 — 2026-05-17

## 核心规则：完整批次执行，不中途汇报

用户明确纠正过两次：
1. "你不用总结啊，你需要将循环2做完呀" — 循环1完成后立即总结错误
2. "还没做完吗" — 在6遍代入中期停顿汇报，用户不耐烦

### 适用规则
- 所有分批次任务（2循环×3序列=6遍代入, multi-pass的N路并行等）
- 必须在完整批次全部完成后才给出结论性输出
- 批次之间不插入总结、汇报、或任何用户可见的中间信息

### 例外
- 如果执行过程中发现致命错误（工具调用失败、数据损坏等），应暂停并报告
- 正常执行中的状态变更（todo更新、中间文件写入）不影响用户的无声执行

### 原因
用户是"不稳定熵增主体"——文字表述不一定100%精确。当用户表述和执行规范文档不一致时，需要用自己的判断找到真正服务进化目的的路径，而非机械遵守其一。
