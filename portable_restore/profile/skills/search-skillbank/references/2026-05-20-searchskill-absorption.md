# SearchSkill 吸收记录：Select-Read-Act + SkillBank

## 来源

用户提供《超级进化.md》，核心内容对应 SearchSkill / SkillBank / Select-Read-Act 检索技能化思想。

## 本次吸收判断

- 吸收：检索技能化、SkillBank、Select-Read-Act、Review/Evolve、失败样本反推、技能淘汰。
- 不吸收：立即大改 Hermes 底层、强制 C/Go/Rust、杜绝 Python、写入所有 md、全量融合口号式功能。

## 可复用规则

1. 复杂检索先 Select：判断是事实查证、法律来源、案例检索、时间限定、多跳、交叉验证还是外部学习。
2. 再 Read：读取对应检索技能的查询模板和验收标准。
3. 再 Act：生成精准查询，优先读取原文/官方/权威来源。
4. Review：记录成功路径或失败原因。
5. Evolve：只有重复成功、高成本失败、长期复用三类情况才沉淀技能。

## 验证结果

- 使用 learning-source + cross-verify 两类检索技能验证。
- 精准查询命中并读取原文：SearchSkill: Teaching LLMs to Use Search Tools with Evolving Skill Banks。
- 验证结论：轻量工作流可提升检索路径控制力，适合先在法律检索、开智外部学习、复杂事实核验中试运行。

## 后续升级门槛

只有当该工作流在多个真实任务中稳定减少检索偏移、提升原文读取率、减少重复搜索时，才考虑更深层工程集成。
