## 2. 根因

4 通道 LLM 并行时，**gpt5.5 (chuangagent) + minimax 90s timeout ERROR**，
剩余 deepseek/agnes/mimo 3 通道损失了约一半 prompt 上下文（特别是
FACT_BLOCK 引用段），LLM 自由发挥补全 → 6+ 项案外事实被编造。

**关键观察**：P5 巡视 + P6 审计的 LLM 检查 4 通道 PASS 全部通过，
**完全没发现 v1 起诉状的事实错误**。LLM 自检不可信。

## 3. 强制 mitigation 清单

### 3.2 通道层：长任务避免 gpt5.5

| max_tokens | 推荐通道 | 备注 |
|---|---|---|
| ≤ 2200 | 全 4 通道 | 安全 |
| 2200-3000 | 4 通道 + 预期 1 ERROR | 监控 gpt5.5 |
| 3000-4500 | 3 通道（deepseek+agnes+mimo） | 排除 gpt5.5 |

### 3.3 验证层：程序正则自检（LLM 不可信）

在 `事实与理由` 段写一个 python 自检脚本（regex-based）：

```python
import re
with open("民事起诉状_FINAL_v2.md") as f:
    text = f.read()
fr_match = re.search(r"### 事实与理由\s*\n(.+?)(?=\n### |\n## )", text, re.DOTALL)
fr = fr_match.group(1) if fr_match else text

required_facts = [
    (r"2025年1月12日", "事故时间"),
    (r"P12902024110106000456", "保单号"),
    (r"董战胜", "工亡雇员"),
    (r"30%", "次责比例"),
    (r"800[,\s]?000", "原告诉求80万"),
    (r"240[,\s]?000", "被告已付24万"),
    (r"郑劳人仲案字【2026】802号", "工亡文号"),
    (r"特别约定第13条", "争议条款"),
    (r"河南民邦人力资源服务有限公司", "原告"),
    (r"燕赵财产保险股份有限公司", "被告"),
    (r"推销员", "雇员岗位"),
    (r"河南世通食品有限公司", "派遣单位"),
    (r"2025年1月1日", "保单起始"),
    (r"2025年1月31日", "保单结束"),
    (r"560[,\s]?000", "争议金额56万"),
]
forbidden_facts = [
    (r"高血压", "v1 错误：高血压既往病史"),
    (r"工地", "v1 错误：工地事故"),
    (r"3月15日", "v1 错误日期"),
    (r"2025年3月", "v1 错误月份"),
    (r"建筑材料", "v1 错误：运送建筑材料"),
    (r"重型自卸货车", "v1 错误：重型自卸货车"),
    (r"2024年11月", "v1 错误：保单起始"),
    (r"2025年10月", "v1 错误：保单结束"),
]

missing = [n for pat, n in required_facts if not re.search(pat, fr)]
forbidden_hit = [n for pat, n in forbidden_facts if re.search(pat, fr)]
assert not missing, f"缺少真实事实: {missing}"
assert not forbidden_hit, f"发现虚假事实: {forbidden_hit}"
print("✅ P7 FINAL v2 程序自检全部通过")
```

**为何不用 LLM 自检**：P5 巡视 + P6 审计 LLM 已实证会通过幻觉稿。
**程序自检** 是确定性的、零漏检的、可重跑的。

### 3.4 失败处理

| 失败模式 | 处理 |
|---|---|
| 1-2 项 required 缺失 | patch 补全（手工 edit） |
| ≥3 项缺失 或 任意 forbidden 命中 | 标记 `.OBSOLETE_事实错误` + 重生成 |
| P5/P6 巡视 LLM 通过但程序自检失败 | **以程序自检为准** → 标注 LLM 巡视降权 |
