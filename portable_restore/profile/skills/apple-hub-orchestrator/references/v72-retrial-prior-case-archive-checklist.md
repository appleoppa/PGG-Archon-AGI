# V7.2 再审/前案边界案件启动与归档清单

触发：用户给出本地再审、申诉、重复起诉、既判力、一事不再理、前案裁判边界案件目录，并要求“读取所有文件/开始办案/调档”。

## 一、启动顺序

1. 扫描用户给定目录全部文件，排除 `.DS_Store` 等系统噪音；
2. 对 PDF、DOC、DOCX、图片分别提取文本/OCR，生成材料提取汇总和 manifest；
3. 案件管理中心编号建档，复制原始材料；
4. 生成 V7.2 标准案件任务包和案件项目计划卡；
5. 证据管理部先做证据快筛，识别 P0/P1/P2 缺口；
6. 再进入主办部门、律法支持、案件推演、智脑知识并行初评；
7. 文书只能在证据 Gate 允许范围内生成。

## 二、关键前案门禁

涉及前案边界时，以下材料应前置校验：

| 材料 | 等级 | 规则 |
|---|---|---|
| 前案完整裁判文书 | P0 | 缺失则不得生成正式提交版 |
| 前案诉辩材料 | P1 | 缺失则标注风险 |
| 前案证据目录/质证/庭审笔录 | P1 | 用于判断是否已实质审理 |
| 生效/履行材料 | P1 | 用于判断既判力和履行边界 |

缺 P0 时，Gate 固定为：

```text
Hold for Submission / Conditional Go for Internal Revision
```

## 三、文书边界

- 可生成：内部初步分析、内部修订稿、补证清单、类案检索方向、风险推演。
- 不可生成：正式提交版、终版、可提交结论、审计通过结论。
- 文首必须标明“内部版/待补材料/未达提交状态”。

## 四、调档规则

用户说“暂时归档/以后补材料/调档”时：

1. 更新 `meta.json`、台账、案件速记；
2. 在 `归档/` 写《阶段性归档记录》《案件运行指标表》《案件流程复盘卡》；
3. 建立《调档触发清单》；
4. 调档后第一动作：更新材料清单 → 重做证据快筛 → 必要时生成前后案三要件对比表；
5. 不得直接打开旧文书继续改。
