# V7.2民事再审案件启动清单

适用场景：用户给出一个本地案件目录并说“开始办案/读取所有文件/再审案件”等，需要按苹果中枢办案系统启动正式案件。

## 操作顺序

1. 加载并遵守 `apple-hub-orchestrator`、`agent-cms`、主办部门、证据管理、律法支持、案件推演、OCR文档类技能。
2. 先扫描用户给出的目录，排除 `.DS_Store` 等系统文件，记录全部材料路径、大小、类型。
3. 提取文本：
   - PDF：先用 `pymupdf` 取文本；有效字符过少时转图片OCR；
   - DOC/DOCX：DOC可用 `textutil`，DOCX用 `python-docx`；若DOCX正文为空，要检查并OCR `word/media/` 内嵌图片；
   - JPG/PNG：用OCR提取，并标注“手续/身份类OCR需人工核验”。
4. 案件管理中心编号建档，复制原始材料到办案库 `原始材料/`，保存全文提取汇总和manifest到 `过程材料/`。
5. 生成《案件受理与派发单》，更新 `meta.json`、台账、案件速记。
6. 按V7.2先生成《标准案件任务包》和《案件项目计划卡》，再进入部门处理。
7. 先做证据管理部“证据门禁快筛”，再做主办部门初评、律法依据初核、案件推演。
8. 汇报时必须区分：已开案、已读取、内部初评完成、待补证、未审计、未终版。

## 民事再审案件特别注意

- 再审类案件通常需要把“原一审/二审裁判链”和“关键前案”作为P0材料。
- 如果争议涉及“重复起诉”，必须取得前案完整判决，不能只凭后案判决摘录替代。
- 二审若同时使用程序性理由（重复起诉）和实体改判，要单独标记为可能的裁判逻辑混杂攻击点。
- 缺前案全文、一二审证据目录、庭审/质证笔录时，只能进入内部预分析，不得标注“可提交终版”。

## 汇报状态建议

使用：

```text
部分完成-已启动办案-内部初评完成-待补关键材料
```

不要使用：

```text
办结 / 终版 / 可提交
```

除非已经完成律法正式复核、巡视终审和审计门禁。
