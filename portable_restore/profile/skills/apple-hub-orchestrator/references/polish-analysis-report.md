# 分析报告打磨：草案→终版对外版本

## 背景

苹果哥要求：交付的终版分析报告必须是"对外可直接使用的完整分析报告"，不含任何内部过程痕迹。01_分析报告草案.md需打磨后才能进入交付物清单。

## 需要移除的内容

| 痕迹类型 | 原文示例 | 处理方式 |
|---------|---------|---------|
| 文档状态标注 | "草案"、"草稿/待审核" | 删除或改为"V1.0" |
| 置信度评级 | "置信度：高/中/低" | 删除评级，内容保留为客观分析 |
| 内部策略注释 | "注意：此处引用《民法典》第151条仅作为说理辅助论据" | 删除或改写为正文分析 |
| 内部流程提示 | "不宜单独作为诉讼请求的法律依据" | 改写为风险提示或策略说明 |
| 内部标签 | "主攻/备攻/兜底" | 保留多层策略结构，改为"第一层/第二层/第三层"等中性表述 |
| 报告编制信息中的状态 | "状态：草稿/待审核" | 改为版本号 |
| 报告编号中的内部标记 | "DRAFT" | 去掉DRAFT后缀 |

## 需要保留的内容

- 全部法律分析（保留核心论证）
- 法律依据清单（无需改动）
- 证据清单（无需改动）
- 风险提示（改写为客观语言）
- 诉讼策略建议（去掉内部策略注释）
- 综合评估（去掉置信度分数，保留结论）
- 附录（法律条文全文，无需改动）

## 案例：018号报告打磨结果

原始标题：`018号 民邦人力雇主责任险案——法律分析报告草案`
终版标题：`018号 民邦人力雇主责任险案——法律分析报告`

去除项目：置信度评级×6处、内部策略注释×3处、报告编制信息中的"草稿/待审核"状态、报告编号DRAFT后缀、内部指引性语言。

保留结构：案件基本信息→争议焦点分析（4项）→论证方案（3层递进）→法律依据清单（16项）→证据清单（18项）→风险提示（6项）→诉讼策略（5项）→综合评估→附录（关键法律条文）
