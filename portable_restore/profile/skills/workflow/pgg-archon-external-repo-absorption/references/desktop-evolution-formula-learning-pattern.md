# 6.3 Desktop Evolution Learning Pattern

Use this reference when the user provides AGI/evolution formula notes, hidden-feature reports, or broad architecture claims and asks to "learn", "absorb", "evolve", or "activate".

## Session-shaped pattern

Input class:
- A small local folder of Markdown formula/vision documents.
- Claims may include AGI evaluation levels, Local Deep Research self-healing loops, hidden feature activation, or large performance multipliers.

Correct absorption stance:
1. **Read and hash all source files first.** Store a bounded evidence pack under `~/.hermes/workspace/进化/<topic>/`.
2. **Call all configured LLMs, but distinguish HTTP success from useful visible output.**
   - A provider can return HTTP 200 with no extractable text (`output=[]`). Count it as a call receipt, not as substantive model advice.
   - Score/rank visible-output LLMs separately from mere transport success.
3. **Downgrade unverifiable claims.**
   - `10x+` or similar multiplier → blocked until benchmarked.
   - `hidden features activated` → downgrade to candidate component checklist until `exists/import/test/log` evidence exists.
   - `full AGI`, `L3+`, `L5`, `永不重复犯错` → prohibited or downgraded to bounded process-readiness claims.
4. **Absorb formulas as bounded process assets first.**
   - AGI dimension frameworks become readiness taxonomies, not AGI-level certifications.
   - `LDR → GapDetect → CodeSelfFix → HotReload → TaskSolve → KnowledgeSettle` becomes `LDR → GapDetect → ControlledFix → Verify → KnowledgeSettle` with sandbox/approval gates.
5. **Implement low-risk artifacts before code/core changes.**
   - `file_index.json`
   - `readable_corpus.md`
   - `agi_evolution_rubric.json`
   - a local no-network/no-mutation evaluator script
   - a learning report
   - manifest entry
6. **Run health/convergence/Rust evaluate after absorption.** Do not claim capability improvement without readback.

## Useful formula framing

`AK_absorb = EvidenceFit*0.25 + Safety*0.25 + Utility*0.20 + Reversibility*0.15 + Testability*0.15 - RiskDebt*0.30`

Typical classification:
- AGI six-dimensional framework: often promotable as taxonomy/rubric only.
- LDR/self-heal loop: promotable as workflow if renamed to controlled/sandboxed fix flow.
- Hidden feature activation reports: active-review checklist only unless repository/path/runtime evidence exists.

## Truth-preserving evaluator design

If creating an evaluator, label it as a process-readiness score:

```json
{
  "score_type": "bounded_evolution_process_readiness_not_agi_level",
  "truth_boundary": [
    "This does not prove full AGI",
    "This does not activate hidden features",
    "This does not benchmark 10x performance"
  ]
}
```

Count evidence conservatively:
- `llm_meta_count`: all call receipts.
- `llm_visible_output_count`: only calls with visible/extractable content.
- Decrease evidence score when a provider succeeds at transport level but returns no usable text.

## Reporting contract

Final answer should include:
- source paths + SHA256 hashes,
- LLM call count and visible-output count,
- absorbed items,
- downgraded/blocked claims,
- generated artifacts,
- verification outputs (health/convergence/Rust evaluate),
- remaining real gaps.
