## Critical Gates

- Entry prerequisite order gate: load matching skills/specs before task body.
- Full system scan before sequence substitution: skills_list plus 10-15 relevant skill reads and key rule files.
- Three sequences must produce real substitution points, triggers, exposed signals, observations.
- Weaknesses must arise from substitution or real execution failure, not be preselected and backfilled.
- External learning must happen after weakness exposure and change the repair action.
- Runner/cron/no_agent completion must verify business artifacts and truth gates, not just exit_code/status. For evolution pipelines, `status=ok` is only transport success; if apply is still dry-run, candidates remain reviewable, training is only dataset packaging, or sample/label balance makes accuracy meaningless, report **部分完成/未完整进化** rather than “正常完成”.
- Active `开智/` directories should not retain per-round raw folders long-term.

## Known Traps

- Audit order error: quality checks before gene submission omit the new gene.
- Execution-will failure: when a reversible technical repair is >75% confidence, execute rather than labeling it “needs user decision.”
- Self-score misuse: >75% rule applies to whether to ask the user, not whether to continue scheduled rounds.
- Single-model loop: evolution should use cross-model verification or model diversity in audit/arbitration steps.

## Completion Gate

A valid cycle needs: explainable weakness source, real external learning, changed repair action, landed changes, verified DB record, evidence/risk boundary, self-driven report, and real-task migration.
