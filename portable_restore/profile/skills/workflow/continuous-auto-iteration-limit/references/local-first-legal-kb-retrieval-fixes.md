# 本地法律知识库检索修复模式

## 背景

本会话中，法律知识库从外部公开源优先纠偏为 local-first（本地优先）：用户本地已有官方指导案例库与法律法规库，应作为主库；外部公开源只作 supplemental（补充来源）。

## 可复用修复模式

### 1. 先定位本地主库

优先检查：

- `/Users/appleoppa/.hermes/workspace/智库/案例库`
- `/Users/appleoppa/.hermes/workspace/智库/法律法规库`

不要用外部网页样例代替本地主库。

### 2. 目录/索引误命中处理

症状：查询如“劳动争议司法解释”时，`目录索引.md`、`索引清单.json` 排在真实法规前。

修复：

- 对标题/路径含 `目录索引`、`索引清单`、`完整目录`、`index` 的文档降权；
- 索引类只做导航兜底，不参与正文泛匹配刷分；
- 回归时单独保留此类 query。

### 3. 中文法律查询分词

症状：`劳动争议司法解释` 不能命中 `最高人民法院关于审理劳动争议案件适用法律问题的解释`。

修复：

- 保留原始词；
- 补 2-4 字 CJK ngram；
- 补常见法律复合词：`劳动争议`、`合同编`、`民事诉讼`、`企业破产`、`技术秘密`、`非法占用耕地`；
- 将用户说的 `司法解释` 兼容为标题中的 `解释`。

### 4. 标题 compact 匹配

症状：书名号、括号、空格导致标题短语匹配失败。

修复：对 query/title 去除空白、书名号、括号、引号、标点后再做短语匹配。

### 5. 日期版本加权

症状：同一法律不同修正日期互相抢位，如公司法、劳动法、建筑法。

修复：从查询提取 `YYYYMMDD`，若标题含该日期强加权；正文含日期弱加权。

### 6. 评测假阴性避免

不要把完整标题截断为 `中华人民共和国保` 这类无效短查询。完整标题查询应保留完整标题；核心词查询不要强制绑定日期。

### 7. 条文级检索

对 `law_articles_local` 建 CLI 时：

- 跳过索引类来源；
- `--law-title` + `--article-no` 应强约束；
- 精确法律本文（如 `中华人民共和国公司法 20231229`）应优先于 `最高人民法院关于适用...解释/规定`；
- 回归样例至少包括民法典第577条、第509条、第186条、公司法第一条、企业破产法第一条。

### 8. 条文表质量清理

`law_articles_local` 可能从 `索引清单.json`、`目录索引.md` 误抽“第X条”片段。处理方式：

1. 先统计：`title like '%索引%' or title like '%目录%' or source_path like '%.json' or source_path like '%目录索引.md'`；
2. 清理前先 `CREATE TABLE law_articles_local_indexlike_backup_时间戳 AS SELECT ...`；
3. 再 DELETE 对应误抽行；
4. 验证 `post_index_like = 0`、备份数量等于删除数量；
5. 报告中写明可回滚备份表名。

### 9. 统一 CLI 入口

建议保留兼容入口，同时增加统一入口：

```bash
legal-kb doc --summary --collection 法律法规库 "劳动争议司法解释"
legal-kb article --summary --law-title "中华人民共和国民法典" --article-no "第五百七十七条" "违约责任"
```

统一入口应同时回归：

- 法规文档检索；
- 指导案例检索；
- 民法典条文检索；
- 公司法条文检索。

## 验证门槛

- 修复前失败项必须重跑；
- 小回归集达到 100%；
- 扩展回归集要读回 JSON 中 total/passed/pass_rate；
- 清理类动作必须有备份表/备份文件和清理后计数；
- 报告不能只写“已修复”，必须列出失败原因、修复点、回归数字和证据文件。
