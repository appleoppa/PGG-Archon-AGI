### 🔍 检索方法论

#### 关键词检索
1. **确定核心词汇**：法律术语/日常用语/英文缩写
2. **构建检索式**：AND/OR/NOT组合
3. **限定条件**：年份/法院/当事人类型/案由
4. **迭代优化**：根据初步结果调整检索策略

#### 案例检索步骤
```
1. 明确检索目的（找支持/找反驳/找参考）
2. 确定案件类型（案由/法院层级/地域）
3. 选择数据库（官方+商业组合）
4. 执行检索（标题/全文/裁判要点）
5. 筛选结果（相似度/时效性/权威性）
6. 提炼规则（裁判要旨/法律依据）
7. 整理报告（检索策略+结果+适用建议）
```

## 第三方审计与法律 AGI 验证

当用户要求用真实法律数据、案件流程和第三方 LLM 审计来验证法律 AI / AGI 能力时，使用 `references/legal-mimo-audit-workflow.md`：

- MIMO `mimo_v25_pro_auditor` 仅作为第三方审计 LLM，不作为主办案模型。
- 先验证权威法律源 seed registry，再搭建 case intake → evidence → research → strategy → drafting → inspection → auditor 全流程 Agent。
- 未核验法条原文、案例原文和现行有效状态前，只能出内部草稿，不得对外交付。
- "权威源种子已验证"不得说成"国内/全球最强全量法律法规案件数据库已完成"。
- **实测纪律**：`official_text_extraction` 等能力评分必须用真实文件实测；工具存在 ≠ 工具可用；实测打脸时必须维持原分，不得美化。

- **实测教训 Phase 211**：`official_text_extraction` 从 Phase 208 的 55 分（仅 URL 可达性）出发，Phase 211 用 pdftotext 测试扫描型 PDF 导致误判为 0% 成功（方法错误，非工具失败），后用 `pdftoppm -r 400 -png + tesseract chi_sim+eng --psm 6` 实测发现扫描型 PDF 成功率 75%（3/4），纠正后维持 55 分。Phase 212 确认该管道后提升至 70 分。

- **MiniMax 降级审计**：当 MIMO (`mimo_v25_pro_auditor`) 被 macOS proxy 阻断时，按用户指令降级使用 MiniMax CN (`MINIMAX_CN_API_KEY`) 执行审计，结果同样有效。
