# macOS / Hermes 文件清理追加模式

本参考来自一次 Hermes 工作区低风险清理，适用于后续类似文件治理任务。

## 可复用模式

1. 先读治理入口：`ACTIVE_MANIFEST.md`、`CACHE_POLICY.md`、文件体系清理日志等，确认活跃资产和历史资产边界。
2. 低风险清理优先用“可逆归档”而非直接删除：
   - 建 `workspace/存档/file-cleanup-<timestamp>/moved/`；
   - 迁移前记录源路径、目标路径、类型、大小、时间、权限、hash；
   - 写 `MANIFEST.json`；
   - 操作后复查归档存在、源范围干净、保护资产仍存在。
3. `.DS_Store` 属于系统可再生噪声：
   - 可从 Hermes、Desktop、Downloads、packages 等低风险范围归档清理；
   - 避免扫入 `Library` 等系统应用目录，除非用户明确要求；
   - Finder 可能在验证时重新生成，发现后追加归档并再次复查到 0。
4. `workspace/tmp`、`workspace/tmp_extract` 等临时入口：
   - 文件内容可归档清空；
   - 目录本身可保留为空，避免后续工具因入口缺失而重建或报错；
   - 汇报时写明“目录保留、文件数为 0”，不要把源目录仍存在当成失败。
5. 受保护资产验证至少覆盖：
   - Hermes 主配置、密钥环境、主状态库、会话库；
   - workspace；
   - 进化基因库；
   - 办案库；
   - 治理入口文件。

## 汇报字段

- 当前状态：完成 / 部分完成 / 证据不足。
- 清理方式：归档 / 删除 / 权限止血。
- 数量：清理项、归档文件数、体积。
- 证据：归档目录、`MANIFEST.json`、治理日志。
- 验证：残留数、临时目录文件数、保护资产存在性。
- 风险边界：哪些没有动，哪些只是保留为空目录。
