# Session note: memory/profile/identity consolidation and evidence-gate repair

Use this reference when a future session involves整理长期记忆、用户画像、自我描述、规则、或开智循环完成标准。

## Durable lessons from this session

- Memory confusion can induce false completion claims, but the deeper failure is completion judgment: treating artifacts, scripts, scores, or status fields as proof.
- Evidence downgrade is only an honesty mechanism; it is not the objective and not a stopping point.
- Gate failure should trigger repair-first behavior: identify cause, judge repairability, apply the 75% self-approval rule, repair, rerun the same gate, and continue operation.
- Only stop for the user when the action is below 75%, irreversible, outside authorization, likely to leak secrets, materially destructive, or objectively blocked by missing external conditions.
- For memory/profile/self-description cleanup: archive first, rewrite current-effective files, remove stale progress/old completion claims, then verify rewritten files contain current rules and no stale conflict signals.

## Recommended workflow

1. Identify the files that govern the class of behavior: memory, user profile, identity/self-description, workspace rules, and any loaded skill for the task class.
2. Archive originals into a dated workspace archive before overwriting.
3. Rewrite current-effective files around stable facts, current hard rules, and verified user preferences.
4. Remove transient progress, old cycle numbers, old model state, old “completed” claims, and rules later overturned by user corrections.
5. For execution-gate failures, persist the repair-first rule in all applicable layers: rule file, memory, relevant skill, and any整改/reference document.
6. Verify by checking that each target contains the core rule terms and no known stale conflict terms.

## Completion standard

The update is complete only after read-back or equivalent verification confirms:

- Current rule/policy files include “downgrade is not the goal,” “75%,” “self-repair,” “rerun,” and “normal operation” equivalents.
- Relevant skills include the same operational rule, not only memory.
- Any supporting整改/reference file is linked or at least stored under the class-level umbrella for future reuse.
