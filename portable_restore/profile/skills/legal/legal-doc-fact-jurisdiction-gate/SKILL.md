# 法律文书事实核验与管辖核验门禁

## Trigger

用于任何法律文书进入“FINAL / 定稿 / 可提交 / 对外交付”前，尤其是：

- 民事起诉状、答辩状、代理词、法律意见书；
- 多 LLM 协作生成的文书；
- 用户已指出或系统曾发现事实幻觉、法院管辖错误、类案真实性风险的案件。

## 核心原则

1. **源材料优先**：日期、金额、主体、案号/保单号、事故经过、拒赔理由、法律关系，必须能从客户材料或证据目录回溯。
2. **正文强拦截、附录可留痕**：正文不得残留案外事实；附录/修正说明可以保留旧错误用于审计，但必须明确标为错误或修正记录。
3. **管辖前置核验**：诉状定稿前必须检查级别管辖、地域管辖、专属管辖、协议管辖、标的额标准、法院名称精确性。
4. **类案真实性标注**：未真实检索到的类案不得写成真实案号；只能标注为 `[合理构造示例·非真实案号]`，外部提交前必须替换为真实判例或删除。
5. **门禁不 PASS，不得称外部可提交**。

## 最低核验清单

### 事实核验

- 当事人名称、角色；
- 日期：事故日、投保日、保险期间、付款日、拒赔/理赔日；
- 金额：诉请金额、已付金额、争议金额、计算公式；
- 编号：保单号、案号/仲裁号、身份证号等；
- 事故经过：地点、原因、责任比例、是否工伤/工亡；
- 拒赔理由：必须与材料一致；
- 禁止出现源材料没有的病史、地点、合同期间、事故性质。

### 管辖核验

- 级别管辖：标的额和案件性质是否支持中级法院/专门法院；
- 地域管辖：被告住所地、合同履行地、保险标的物所在地、保险事故发生地；
- 专属管辖：是否涉及不动产、港口、继承等专属事项；
- 协议管辖：合同/保单是否有有效管辖条款；
- 法院名称：如信息不足，先写“有管辖权的基层人民法院”，不得硬写具体法院。

## 本案教训（PGG-MS-20260605-0006）

- v1 起诉状曾出现重大事实幻觉：事故时间、事故地点、高血压既往病史、保单期间等均错误。
- FINAL v2 事实核验通过后，仍残留“郑州市中级人民法院”管辖错误。
- 修正方案：改为“有管辖权的基层人民法院”，并要求主办律师立案前按被告住所地/保险标的物所在地/合同履行地等确定具体法院。
- Rust 门禁首次实跑误杀了附录中的修正记录，随后修成“正文强拦截、附录/示例留痕不误杀”。

## 报告口径

- `PASS`：事实残留和管辖显性问题未命中；仍需律师复核。
- `WARN`：存在需要人工确认的类案、管辖或源事实覆盖风险。
- `BLOCK`：不得外部提交；必须修正后复跑。

## 边界

该门禁是程序化质量控制，不等于法院受理确认，不替代律师对管辖、事实、证据、法律依据的最终判断。
