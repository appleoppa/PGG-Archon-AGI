# Self Reflection — Compact

## Trigger

Use for periodic review of sessions to extract actionable insights, recurring errors, user preferences and workflow improvements.

## Workflow

1. Search session logs for repeated pattern or correction.
2. Separate durable facts, procedures, temporary outcomes and noise.
3. Save user/environment facts to memory only if stable.
4. Save reusable procedures as skills.
5. Do not store stale task progress.

## Output

`insight`, `evidence`, `action`, `memory_or_skill`, `boundary`.
